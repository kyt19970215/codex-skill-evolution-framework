# Project Sources

Use this file for project-specific public source order and search hygiene.

## Source Priority

1. Current local project evidence.
2. Official documentation and release notes.
3. GitHub repositories, issues, discussions, and maintainer replies.
4. Standards/specifications, when relevant.
5. Forums, blogs, or videos as hypotheses only.

## Search Hygiene

- Search public component names and generic error fragments.
- Do not search private local paths, account names, complete logs, or proprietary details.
- Return to local verification before concluding.
