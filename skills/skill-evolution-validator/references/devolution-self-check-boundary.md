# Rule-Maintenance Boundary

When a user asks to inspect whether the evolution system has become bloated or too narrow, this validator may collect evidence first. `skill-evolution-core` still owns any decision to relax, merge, archive, or remove rules.

Never treat a health report as permission to weaken privacy, publication, deletion, account, payment, security, or other high-impact boundaries.

If the current request already authorizes repair, the report should emit a structured handoff to `skill-evolution-core` and `skill-evolution-router`. They own edits, ledger updates, and the full validation rerun. The validator itself remains report-only.

Evidence gaps such as missing real Hook events, unlabeled outcomes, or intentional installed/public divergence remain review-only. Do not convert them into automatic file edits merely because repair is authorized.
