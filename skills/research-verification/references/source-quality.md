# Source Quality

## Strong Sources

- Official documentation.
- Official release notes and changelogs.
- Source code in the official repository.
- GitHub issues/discussions with maintainer participation.
- Standards or specifications.

## Medium Sources

- Stack Overflow answers with reproducible examples.
- Community forum posts with logs and version numbers.
- Technical blog posts that cite upstream sources.

## Weak Sources

- Old blog posts without version dates.
- Videos without source links.
- Forum comments that only report anecdotes.
- AI-generated pages.

## Verification Reminder

External evidence suggests a hypothesis. Local tests decide whether it applies to the user's environment.
