# Research Failure Learning

Use this file for durable lessons about external research, current-version verification, public error lookup, API behavior, dependency releases, browser/driver behavior, install/upgrade issues, and stale documentation.

## Add A Rule When

- A public claim, old note, or memory was stale and current docs changed the conclusion.
- A tool/API/dependency version mismatch caused a failed attempt.
- A search source looked authoritative but was weaker than an official doc, changelog, GitHub issue, or maintainer reply.
- A local fix was only confirmed after returning from external sources to local verification.

## Do Not Add

- Raw search logs.
- Private project paths or proprietary code.
- A rule based only on a forum claim that was not locally verified.
- A broad "always search more" rule when a narrower source-quality rule would do.

- Better source/check:
