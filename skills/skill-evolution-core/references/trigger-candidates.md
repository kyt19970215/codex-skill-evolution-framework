# Trigger Candidates

Use this file as a sanitized ledger for trigger-word learning. Do not store full user prompts, private project details, secrets, account data, local paths, logs, or raw transcripts.

## Candidate Entry Template

```markdown
## Candidate: <short trigger>

- Count: 1
- Destination:
- Status: watch
- Evidence:
  - YYYY-MM-DD: <sanitized phrase> -> <routing outcome>
- Promotion note:
```

Start with no real entries. Each installation builds its own local ledger from sanitized evidence.
